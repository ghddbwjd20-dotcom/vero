## 📝 콘텐츠 관리

### Sanity CMS 접속

1. [Sanity Studio](https://sanity.io/manage)에 접속
2. 프로젝트 선택
3. Studio 실행

### 기본 콘텐츠 수정

#### 회사 정보 수정
- **위치**: Sanity Studio → Settings → Company Info
- **수정 가능 항목**:
  - 회사명
  - 연락처 정보
  - 주소
  - 소개 문구

#### 메인 페이지 콘텐츠
- **위치**: Sanity Studio → Pages → Home
- **수정 가능 항목**:
  - 히어로 섹션 텍스트
  - 철학 섹션 내용
  - 프로세스 설명
  - 신뢰 포인트

## 🎨 포트폴리오 관리

### 새 프로젝트 추가

1. Sanity Studio → Portfolio → Create
2. 필수 정보 입력:
   - **프로젝트명**: 명확하고 간결한 제목
   - **설명**: 한 줄 요약 (SEO에 중요)
   - **카테고리**: 브랜딩, 커머스, B2B, 모바일 등
   - **태그**: 검색 및 필터링용 키워드
   - **연도**: 프로젝트 완료 연도
   - **썸네일**: 1200x800px 권장
   - **갤러리**: 프로젝트 이미지들
   - **문제/해결/성과**: 케이스 스터디 내용
   - **기술 스택**: 사용된 기술들
   - **역할**: 담당 범위
   - **링크**: 라이브 사이트, GitHub 등

### 이미지 최적화 가이드

#### 썸네일 이미지
- **크기**: 1200x800px
- **형식**: JPG 또는 PNG
- **용량**: 500KB 이하
- **압축**: 80-85% 품질

#### 갤러리 이미지
- **크기**: 1920x1080px (16:9 비율)
- **형식**: JPG
- **용량**: 1MB 이하
- **압축**: 85% 품질

### 프로젝트 순서 관리

포트폴리오는 `publishedAt` 날짜 기준으로 정렬됩니다.
- 최신 프로젝트가 상단에 표시
- 날짜를 조정하여 순서 변경 가능

## 📰 공지사항 관리

### 새 공지사항 작성

1. Sanity Studio → News → Create
2. 필수 정보 입력:
   - **제목**: 명확하고 검색 가능한 제목
   - **슬러그**: URL에 사용될 고유 식별자
   - **카테고리**: 업데이트, 프로젝트, 채용, 인사이트 등
   - **태그**: 검색 및 분류용 키워드
   - **썸네일**: 1200x600px 권장
   - **본문**: Markdown 형식으로 작성

### 카테고리 관리

#### 업데이트
- 웹사이트 업데이트
- 서비스 개선 사항
- 새로운 기능 추가

#### 프로젝트
- 프로젝트 완료 소식
- 케이스 스터디
- 성과 공유

#### 채용
- 인재 채용 공고
- 팀 확장 소식
- 채용 프로세스 안내

#### 인사이트
- 업계 트렌드 분석
- 기술 블로그
- 베스트 프랙티스

## 📧 문의 관리

2. **이메일 확인**: 관리자 이메일로 복사본 발송
3. **고객 확인**: 고객 이메일로 접수 확인 발송

### 문의 응답 프로세스

1. **24시간 내**: 초기 응답 (접수 확인)
2. **3일 내**: 상세 상담 일정 조율
3. **1주일 내**: 제안서 또는 견적 제공

### 문의 데이터 관리

- **저장 위치**: Sanity CMS → Contacts
- **보관 기간**: 3년
- **개인정보**: GDPR 및 개인정보보호법 준수

## 🔍 SEO 관리

### 메타데이터 관리

#### 페이지별 메타데이터
- **제목**: 60자 이하, 키워드 포함
- **설명**: 160자 이하, 매력적인 요약
- **키워드**: 관련 키워드 3-5개
- **OG 이미지**: 1200x630px

#### 키워드 전략
- **주요 키워드**: 웹 에이전시, 웹 개발, UI/UX
- **롱테일 키워드**: "웹 에이전시 추천", "브랜딩 디자인"
- **지역 키워드**: "서울 웹 에이전시", "강남 웹 개발"

### 사이트맵 관리

- **자동 생성**: `/sitemap.xml`
- **업데이트**: 콘텐츠 변경 시 자동 갱신
- **제출**: Google Search Console에 등록

### Google Search Console 설정

1. [Google Search Console](https://search.google.com/search-console) 접속
2. 속성 추가 (도메인 또는 URL 접두어)
3. 소유권 확인
4. 사이트맵 제출
5. 색인 생성 요청

## 💾 백업 및 복구

### 자동 백업

#### Sanity CMS
- **빈도**: 일일 자동 백업
- **보관 기간**: 30일
- **복구**: Sanity Dashboard → Backups

#### 코드 저장소
- **위치**: GitHub
- **보호**: Branch protection rules
- **백업**: 별도 저장소에 미러링

### 수동 백업

#### 콘텐츠 백업
```bash
# Sanity 데이터 내보내기
npx sanity dataset export production backup-$(date +%Y%m%d).tar.gz
```

#### 환경 설정 백업
- `.env` 파일 암호화 저장
- Vercel 환경 변수 스크린샷
- 도메인 설정 문서화

### 복구 절차

#### 콘텐츠 복구
1. Sanity Dashboard → Backups
2. 복구할 백업 선택
3. 데이터셋 복원 실행

#### 사이트 복구
1. GitHub에서 이전 커밋 확인
